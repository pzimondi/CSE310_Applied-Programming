About TaskFlow

How it was built and why.

The Project

TaskFlow is a full-stack task management web application built as part of CSE 310 — Applied Programming at Brigham Young University–Idaho.

The goal was to learn how a real web server works from the ground up — how the front end and back end communicate through a REST API, and how to persist data in a cloud database.

Tech Stack

  • Node.js — JavaScript runtime for the server
  • Express — handles routing and middleware
  • MongoDB Atlas — cloud database for task storage
  • Mongoose — schema validation and DB queries
  • Vanilla JS (fetch API) — client-side REST calls

Features

  • Add tasks with title and priority level
  • Mark tasks complete — toggles back if needed
  • Delete tasks permanently
  • Filter: All / Pending / Done
  • Persistent storage via MongoDB Atlas
  • Live stats fetched from the server

What I Learned

This was my first time building for a browser. The biggest shift from Python was asynchronous JavaScript — async/await and fetch() don't block the page while waiting for a server response.

CORS was the first stumbling block. Installing the cors npm package and calling app.use(cors()) before any endpoints solved it immediately.
